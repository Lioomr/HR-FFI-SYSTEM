# Asset Management

Use this when changing asset CRUD, assignment/return flows, damage reports, label printing, scanner lookup, or asset PDFs.

## Core Models

- `Asset` uses `asset_code` generated from `AssetCodeSequence` prefixes: `VEH`, `LAP`, `AST`.
- `AssetAssignment` has at most one active assignment per asset through `unique_active_assignment_per_asset`.
- `AssetDamageReport` and `AssetReturnRequest` are linked to `Asset` and participate in approval/history workflows.
- `PrintedLabelJob` stores label print history with `company`, `created_by`, `asset_count`, `paper_size`, `pdf_file`, and `asset_codes`.

## Label Printing And Scanner Lookup

- Label PDFs are generated by `Backend/assets/services/label_pdf.py`.
- Label jobs save PDFs in `PrivateUploadStorage` under `assets/labels/`.
- Supported paper sizes are `50X30`, `40X30`, `60X40`, and `A4_GRID`.
- Labels include QR, Code128, human-readable `asset_code`, and English asset name.
- Scanner lookup is `GET /api/assets/lookup/?code=<asset_code>` and returns the asset, active assignment, latest 3 damage reports, and latest 3 return requests.

## Guardrails

- **Never bulk-print labels from `Asset.objects.filter(id__in=...)` directly** - this can leak other-company assets into a label batch. Use `filter_queryset_by_company_scope(Asset.objects..., request).filter(id__in=asset_ids)` and reject the whole batch if any requested ID is missing after scoping.
- All label-printing and scanner-lookup endpoints are HR-only: `IsHRManagerOrSystemAdmin`.
- Label print actions must audit `asset_label_printed`.
- Re-download endpoints must serve the stored file through an authenticated `FileResponse`, never a public URL.
- `AssetLookupSerializer` receives an `Asset` instance directly; render its `asset` field with `AssetSerializer(obj, context=self.context).data`.

## Tests To Run

Backend:

```bash
cd Backend && pytest assets/tests.py::AssetLabelAndLookupTests assets/tests_pdf.py
cd Backend && python manage.py check
cd Backend && python manage.py makemigrations --check assets
```

Docker:

```bash
docker compose -f docker-compose.dev.yml exec -T backend pytest assets/tests.py::AssetLabelAndLookupTests assets/tests_pdf.py
docker compose -f docker-compose.dev.yml exec -T backend python manage.py check
```

Manual preview:

- Print one 50x30 label and confirm the response is `application/pdf` with `X-Label-Job-Id`.
- Print an A4 grid for multiple assets and confirm a `PrintedLabelJob` exists.
- Re-download the saved PDF from `/api/assets/labels/jobs/{id}/pdf/`.
- Lookup an asset code with different casing and confirm the active assignment and recent histories return.
- Switch active company and confirm other-company lookup/print returns 404.
