# Task Execution

Default workflow:
1. Read context and relevant plan files.
2. Inspect existing code before editing.
3. Make focused changes.
4. Update tests or docs when behavior changes.
5. Run targeted validation.
6. Summarize changed files and commands run.

Avoid:
- Unrelated formatting churn.
- Large rewrites without need.
- Changing API names without compatibility handling.
- Deleting user work.
